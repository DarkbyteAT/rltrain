    GradientTransform ¶

    Bases: Protocol

    A single step in the gradient transform pipeline.

    Each transform receives the model (with gradients already populated from loss.backward()), a loss_fn callable that can recompute the loss at a perturbed point, and the original batch.

    Two hooks are available:

    • apply() runs before descend() — for transforms that modify gradients or temporarily perturb parameters (e.g. SAM, ASAM).
    • post_step() runs after descend() — for transforms that operate on the updated parameters (e.g. LAMP noise injection + rollback).

    Concrete classes must implement both methods. Use an empty body (...) for phases the transform does not participate in.
